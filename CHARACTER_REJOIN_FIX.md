# Исправление проблемы повторного входа игрока

## Проблема
При повторном заходе в лобби игрок не мог выбрать своего персонажа, так как он отображался как "Занят". При этом персонаж хоста был доступен для выбора (что неверно).

## Причина
В компоненте `CharacterSelect.tsx`:
- `occupiedCharacterIds` содержал ВСЕ персонажи в лобби
- Не было проверки, какой персонаж принадлежит текущему игроку
- При повторном входе персонаж игрока считался занятым

## Решение

### 1. Добавлено состояние для отслеживания своего персонажа
```typescript
const [myCharacterId, setMyCharacterId] = useState<string | null>(null);
```

### 2. Обновлена функция `fetchCharacters`
Теперь получает `user_session_id` для каждого участника и находит персонажа текущего игрока:
```typescript
const { data: participants } = await supabase
  .from('lobby_participants')
  .select('character_id, user_session_id')
  .eq('lobby_id', roomId);

const myParticipant = participants?.find(p => p.user_session_id === userSessionId);
setMyCharacterId(myParticipant?.character_id || null);
```

### 3. Обновлена логика проверки занятости
```typescript
const isOccupied = occupiedCharacterIds.has(character.id);
const isMyCharacter = character.id === myCharacterId;
const isOccupiedByOther = isOccupied && !isMyCharacter;
```

### 4. Улучшена функция `handleSelectCharacter`
- Разрешает выбор своего персонажа (повторный вход)
- Запрещает выбор персонажа, занятого ДРУГИМ игроком
- При повторном входе обновляет запись вместо создания новой

### 5. Обновлён UI
- Свой персонаж помечается значком "Ваш персонаж" (зелёный)
- Чужой занятый персонаж помечается как "Занят" (красный)
- Кнопка выбора доступна для своего персонажа

## Тестирование

### Сценарий 1: Повторный вход
1. Игрок 1 заходит в лобби с персонажем "Арагорн"
2. Игрок 1 выходит и заходит снова
3. ✅ "Арагорн" отображается как "Ваш персонаж"
4. ✅ Игрок 1 может выбрать "Арагорн"

### Сценарий 2: Выбор персонажа хоста
1. Игрок 1 (хост) заходит с персонажем "Гэндальф"
2. Игрок 2 заходит в лобби
3. ✅ "Гэндальф" отображается как "Занят"
4. ✅ Игрок 2 НЕ может выбрать "Гэндальф"

### Сценарий 3: Смена персонажа
1. Игрок заходит в лобби с персонажем A
2. Выбирает персонажа B (свободного)
3. ✅ Запись обновляется (не создаётся дубликат)
