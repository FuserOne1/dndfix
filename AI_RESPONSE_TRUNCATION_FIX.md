# Исправление обрезанных ответов AI

## Проблема
Ответы от Dungeon Master (AI) выходили обрезанными, неполными.

## Причины

### 1. Ограничение max_tokens
В `ai-orchestrator.ts` было установлено `max_tokens: 1024`, что недостаточно для подробных RPG-сцен.

### 2. Малый контекст истории
В `Chat.tsx` использовались только последние **5 сообщений**, что теряло важный контекст.

### 3. Агрессивная обрезка сообщений
Сообщения длиннее **1500 символов** обрезались, что могло терять важную информацию.

### 4. Отсутствие указаний в промпте
В SYSTEM_PROMPT не было явного требования писать подробные ответы.

## Решение

### 1. Увеличен max_tokens
```typescript
// Было
max_tokens: 1024

// Стало
max_tokens: 2048  // Увеличено для более полных ответов
```

**Файлы:**
- `src/lib/ai-orchestrator.ts` (2 места: generateResponse и summarizeHistory)

### 2. Увеличен контекст истории
```typescript
// Было
const recentHistory = rpHistory.slice(-5);

// Стало
const recentHistory = rpHistory.slice(-10);  // Последние 10 сообщений
```

**Файл:** `src/components/Chat.tsx`

### 3. Увеличен лимит обрезки сообщений
```typescript
// Было
if (content.length > 1500) {
  content = content.substring(0, 1500) + "...";
}

// Стало
if (content.length > 4000) {
  content = content.substring(0, 4000) + "...";
}
```

**Файл:** `src/components/Chat.tsx`

### 4. Добавлено требование в SYSTEM_PROMPT
```
- **ДЛИНА ОТВЕТА**: Пиши ПОДРОБНЫЕ, атмосферные ответы (300-800 слов). 
  Не обрезай мысли на полуслове. Разрешай сцену полностью, прежде чем 
  предлагать варианты действий.
```

**Файл:** `src/components/Chat.tsx`

## Результат

| Параметр | Было | Стало | Улучшение |
|----------|------|-------|-----------|
| Max tokens | 1024 | 2048 | +100% |
| Контекст сообщений | 5 | 10 | +100% |
| Лимит обрезки | 1500 | 4000 | +167% |

## Тестирование

1. Создать новую игру
2. Написать развёрнутое действие
3. ✅ Ответ AI должен быть подробным (300-800 слов)
4. ✅ Ответ не должен обрываться на полуслове
5. ✅ Контекст предыдущих сообщений должен учитываться

## Примечание

Увеличение max_tokens может привести к росту стоимости запросов. 
Мониторьте использование API через OpenRouter dashboard.
