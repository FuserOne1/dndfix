# Проверка: AI не выводит JSON как код

## Текущая конфигурация

### 1. SYSTEM_PROMPT настроен правильно

```
Когда характеристики персонажа меняются [...] ты ДОЛЖЕН в самом конце своего сообщения добавить скрытый блок JSON в формате:

[STATS_UPDATE:{"name":"Имя персонажа","race":"Раса",...}]

ВАЖНО:
- НИКОГДА не пиши этот блок в коде (не используй обратные кавычки)
- используй ТОЛЬКО формат [STATS_UPDATE:{...}]
- НИКОГДА не пиши ничего после этого блока
- НИКОГДА не упоминай этот блок в тексте для игрока
```

✅ **Правильно:** AI использует формат без обратных кавычек

### 2. Парсинг поддерживает оба формата

```typescript
const statsUpdateMatch = aiText.match(/\[STATS_UPDATE:([\s\S]*?)\]/);
const jsonMatch = aiText.match(/```json\n([\s\S]*?)\n```/);

if (statsUpdateMatch) {
  // Новый формат - приоритет
} else if (jsonMatch) {
  // Старый формат - обратная совместимость
}
```

✅ **Правильно:** Сначала пробуем новый формат

### 3. JSON удаляется из ответа

```typescript
aiText = aiText.replace(/\[STATS_UPDATE:([\s\S]*?)\]/, '').trim();
```

✅ **Правильно:** Блок удаляется перед отправкой в чат

## Поток данных: Обновление карточек

### Проблема, которая была:
> "DM скрипт JSON присылал, а карточки не менялись"

### Причина:
Отсутствовала подписка на изменения `character_stats` в таблице `rooms`.

### Решение:

#### 1. Добавлена подписка в Chat.tsx

```typescript
.on(
  'postgres_changes',
  {
    event: 'UPDATE',
    schema: 'public',
    table: 'rooms',
    filter: `id=eq.${roomId}`,
  },
  (payload) => {
    // Проверяем, изменились ли характеристики персонажей
    const newStats = payload.new.character_stats;
    const oldStats = payload.old.character_stats;
    if (newStats && newStats !== oldStats) {
      console.log('📊 Character stats changed via postgres_changes');
      setCharacterStats(newStats as Record<string, CharacterStats>);
      
      // Обновляем story_summary
      const currentPlayer = Object.keys(newStats).find(name => 
        name === userName || newStats[name]?.name === userName
      );
      if (currentPlayer && newStats[currentPlayer]?.story_summary) {
        setStorySummary(newStats[currentPlayer].story_summary);
      }
    }
  }
)
```

#### 2. Полный поток данных

```
1. Игрок отправляет сообщение
   ↓
2. AI генерирует ответ + [STATS_UPDATE:{...}] в конце
   ↓
3. Chat.tsx парсит [STATS_UPDATE:{...}]
   ↓
4. Извлекает updatedStats
   ↓
5. setCharacterStats() (локально)
   ↓
6. updateRoomStats() (в БД)
   ↓
7. БД обновляет rooms.character_stats
   ↓
8. postgres_changes уведомление
   ↓
9. ВСЕ игроки получают обновление
   ↓
10. setCharacterStats() у всех игроков
   ↓
11. Карточки обновляются у ВСЕХ
```

## Архитектура обновлений

### Кто обновляет карточки?

**DM (AI):**
- Генерирует [STATS_UPDATE:{...}] в конце ответа
- Это **единственный источник правды** для изменений

**Клиент (Chat.tsx):**
- Парсит [STATS_UPDATE:{...}]
- Обновляет локальное состояние
- Сохраняет в БД
- Рассылает всем через postgres_changes

**Дешёвая нейросеть (Gemini):**
- НЕ обновляет карточки напрямую
- Используется только для суммаризации истории
- Может генерировать промпты для изображений

### Почему так?

1. **AI (DM) — источник правды**: Только AI знает контекст и может корректно изменить статы
2. **Клиент — синхронизатор**: Гарантирует, что все игроки видят одинаковые статы
3. **БД — хранилище**: Сохраняет состояние для последующих сессий

## Примечания

### Если карточки не обновляются:

1. **Проверь консоль**:
   - Должно быть: `📊 Found STATS_UPDATE block in new format`
   - Должно быть: `💾 Updating room stats for: ...`
   - Должно быть: `✅ Room stats updated successfully`
   - Должно быть: `📊 Character stats changed via postgres_changes`

2. **Проверь БД**:
   - Открой Supabase → Table Editor → rooms
   - Найди свою комнату
   - Проверь `character_stats` — должен быть JSON с данными

3. **Проверь AI ответ**:
   - В конце ответа должен быть `[STATS_UPDATE:{...}]`
   - Без обратных кавычек
   - Валидный JSON

### Если JSON отображается в чате:

1. AI использует старый формат ```json {...} ```
2. Нужно переобучить AI или отправить новый SYSTEM_PROMPT

### Если карточки обновляются только у хоста:

1. Проверь подписку `postgres_changes`
2. Проверь, что `is_ai_generating` и `character_stats` обновляются в БД
3. Проверь логи: должно быть `Character stats changed via postgres_changes`
