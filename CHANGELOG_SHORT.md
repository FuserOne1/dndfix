## 🎯 ЧТО ИЗМЕНИЛОСЬ

### 1. Архитектура БД
**БЫЛО:**
- `rooms` таблица с JSON `character_stats`
- Статы хранились как JSON по имени игрока
- Путаница между лобби и игровой сессией

**СТАЛО:**
- `game_sessions` — активные игровые сессии
- `game_session_participants` — связь игроков с сессией
- `characters` — единственный источник правды для статов
- Чёткое разделение: **Лобби** (ожидание) vs **Сессия** (игра)

---

### 2. AI больше не пишет JSON
**БЫЛО:**
```
[STATS_UPDATE:{"name":"Арагорн","hp":{"current":7,"max":10}}]
```
AI тратил токены на JSON, который вырезался парсером.

**СТАЛО:**
- AI пишет только текст: *"Вы получили 3 урона, осталось 7 HP"*
- Отдельная модель (Gemini 2.5 Flash) парсит ответ и обновляет статы
- Экономия токенов + красивый вывод

---

### 3. UI с постоянными статами
**БЫЛО:**
- Карточки открывались в модальном окне
- Не видно текущее состояние во время игры

**СТАЛО:**
- Справа сверху **постоянная панель** с характеристиками
- Видно HP, XP, уровень, статы, инвентарь
- Обновляется realtime у всех игроков

---

### 4. Синхронизация
**БЫЛО:**
- Подписка на `rooms.character_stats` (JSON)
- Конфликты при одновременном обновлении

**СТАЛО:**
- Подписка на `characters` (отдельные записи)
- Каждый персонаж обновляется независимо
- Realtime синхронизация между всеми игроками

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### AI Модели
| Роль | Модель | Задача |
|------|--------|--------|
| DM | Claude Sonnet 4.6 | Сюжет, боевые сцены |
| Parser | Gemini 2.5 Flash | Извлечение статов из ответа |
| Summary | Gemini 2.5 Flash | Обновление истории |

### База данных
```
characters (UUID, name, hp, xp, stats, ...)
    ↑
    │
game_session_participants (session_id, character_id, user_id)
    ↑
    │
game_sessions (id, lobby_id, created_by)
    ↑
    │
messages (session_id, sender, content, is_ai)
```

### Скорость
- **Claude ответ:** 10-30 сек
- **Gemini парсинг:** 1-3 сек (асинхронно, не блокирует)
- **Summary:** 1-2 сек (кэшируется)

## ❓ ВОПРОСЫ

Если что-то не работает — смотри консоль браузера (F12) и логи Supabase.

Основные ошибки:
- "session not found" → сессия не создана в БД
- Карточки не обновляются → нет подписки на `characters`
- AI не отвечает → проверить API ключ OpenRouter
