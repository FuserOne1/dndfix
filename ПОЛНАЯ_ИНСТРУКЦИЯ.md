# 🎯 ПОЛНАЯ ИНСТРУКЦИЯ ПО ИСПРАВЛЕНИЮ

## Проблема 1: Персонажи не удаляются
## Проблема 2: Я не отображаюсь в лобби

---

## ✅ РЕШЕНИЕ (один SQL скрипт для всего)

### Скопируйте этот код ПОЛНОСТЬЮ:

```sql
-- Удаляем все старые политики
DROP POLICY IF EXISTS "Anyone can view characters" ON characters;
DROP POLICY IF EXISTS "Anyone can create characters" ON characters;
DROP POLICY IF EXISTS "Anyone can update characters" ON characters;
DROP POLICY IF EXISTS "Anyone can delete characters" ON characters;
DROP POLICY IF EXISTS "Enable read access for all users" ON characters;
DROP POLICY IF EXISTS "Enable insert for all users" ON characters;
DROP POLICY IF EXISTS "Enable update for all users" ON characters;
DROP POLICY IF EXISTS "Enable delete for all users" ON characters;

DROP POLICY IF EXISTS "Anyone can view lobbies" ON lobbies;
DROP POLICY IF EXISTS "Anyone can create lobbies" ON lobbies;
DROP POLICY IF EXISTS "Anyone can update lobbies" ON lobbies;

DROP POLICY IF EXISTS "Anyone can view participants" ON lobby_participants;
DROP POLICY IF EXISTS "Anyone can join lobbies" ON lobby_participants;
DROP POLICY IF EXISTS "Anyone can leave lobbies" ON lobby_participants;

-- Отключаем RLS для всех таблиц
ALTER TABLE characters DISABLE ROW LEVEL SECURITY;
ALTER TABLE lobbies DISABLE ROW LEVEL SECURITY;
ALTER TABLE lobby_participants DISABLE ROW LEVEL SECURITY;

-- Даем полные права
GRANT ALL ON characters TO anon, authenticated, service_role;
GRANT ALL ON lobbies TO anon, authenticated, service_role;
GRANT ALL ON lobby_participants TO anon, authenticated, service_role;

-- Проверка
SELECT 
  tablename,
  rowsecurity as rls_disabled
FROM pg_tables 
WHERE tablename IN ('characters', 'lobbies', 'lobby_participants')
ORDER BY tablename;
```

---

## 📝 Как применить:

1. Откройте Supabase → **SQL Editor**
2. Скопируйте **ВЕСЬ** код выше
3. Вставьте в SQL Editor
4. Нажмите **Run** (или F5)
5. Должно показать "Success"

---

## 🧪 Проверка:

### После выполнения SQL:

1. **Обновите страницу приложения** (F5)
2. **Проверьте удаление персонажей:**
   - Создайте тестового персонажа
   - Нажмите кнопку удаления 🗑️
   - Обновите страницу (F5)
   - Персонаж должен исчезнуть ✅

3. **Проверьте отображение в лобби:**
   - Создайте лобби
   - Выберите персонажа
   - Вы должны появиться в списке участников ✅

## 🔍 Отладка (если всё равно не работает):

### Откройте консоль браузера (F12):

Должны увидеть логи:
```
Fetched participants: [...]
My session ID: session_...
My character: Имя_Персонажа
Found my participant: {...}
```

Если видите `My participant not found in list!` - значит проблема с добавлением в БД.

## 📞 Если не помогло:

Выполните этот SQL и пришлите результат:

```sql
-- Проверка данных
SELECT 
  lp.id,
  lp.lobby_id,
  lp.user_session_id,
  lp.is_ready,
  c.name as character_name
FROM lobby_participants lp
JOIN characters c ON c.id = lp.character_id
ORDER BY lp.joined_at DESC
LIMIT 10;
```
