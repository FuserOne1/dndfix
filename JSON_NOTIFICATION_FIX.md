# Исправление вывода JSON и текстовые уведомления

## Проблема
AI выводил JSON-блок с данными персонажа в формате кода (```json ... ```), который отображался в чате и портил впечатление от игры.

## Решение

### 1. Новый формат скрытого блока

**Было:**
```json
{
  "type": "UPDATE_STATS",
  "stats": {...}
}
```

**Стало:**
```
[STATS_UPDATE:{"name":"Имя","race":"Раса",...}]
```

Новый формат:
- Не отображается как код в Markdown
- Легко парсится регулярным выражением
- Занимает меньше токенов

### 2. Генерация текстовых уведомлений

Система теперь генерирует красивые уведомления об изменениях:

```
━━━
📊 Изменения персонажа
🔴 HP: -3 (10 → 7)
⭐ XP: +50 (0 → 50)
🎯 Уровень повышен: 1 → 2!
━━━
```

**Эмодзи:**
- 🟢 — получение HP (лечение)
- 🔴 — потеря HP (урон)
- ⭐ — получение опыта
- 🎯 — повышение уровня

### 3. Обновлён SYSTEM_PROMPT

Добавлены инструкции для AI:
- Использовать формат `[STATS_UPDATE:{...}]`
- Не упоминать блок в тексте
- Кратко упоминать урон/лечение в тексте
- Поздравлять с повышением уровня

### 4. Обратная совместимость

Код поддерживает оба формата:
1. Сначала пробуем новый `[STATS_UPDATE:{...}]`
2. Если не найден — старый ```json {...} ```

## Изменения в коде

### Chat.tsx

**SYSTEM_PROMPT:**
```typescript
// Было
\`\`\`json
{
  "type": "UPDATE_STATS",
  "stats": {...}
}
\`\`\`

// Стало
[STATS_UPDATE:{"name":"Имя","race":"Раса",...}]
```

**Функция парсинга:**
```typescript
// Поддержка обоих форматов
const statsUpdateMatch = aiText.match(/\[STATS_UPDATE:([\s\S]*?)\]/);
const jsonMatch = aiText.match(/```json\n([\s\S]*?)\n```/);

if (statsUpdateMatch) {
  // Новый формат
} else if (jsonMatch) {
  // Старый формат (обратная совместимость)
}
```

**Генерация уведомлений:**
```typescript
const hpDiff = updatedStats.hp.current - prevStats.hp.current;
const xpDiff = updatedStats.xp - prevStats.xp;
const levelDiff = updatedStats.level - prevStats.level;

const changes: string[] = [];

if (hpDiff !== 0) {
  const hpColor = hpDiff > 0 ? '🟢' : '🔴';
  changes.push(`${hpColor} HP: ${hpSign}${hpDiff} (${prev} → ${curr})`);
}

if (xpDiff > 0) {
  changes.push(`⭐ XP: +${xpDiff} (${prev} → ${curr})`);
}

if (levelDiff > 0) {
  changes.push(`🎯 Уровень повышен: ${prev} → ${curr}!`);
}
```

## Тестирование

### Сценарий 1: Получение урона
1. Игрок атакует монстра
2. Монстр наносит ответный урон (3 урона)
3. ✅ В начале ответа AI: "🔴 HP: -3 (10 → 7)"
4. ✅ JSON-блок не отображается в чате

### Сценарий 2: Лечение
1. Игрок использует зелье лечения
2. Восстанавливает 5 HP
3. ✅ В начале ответа AI: "🟢 HP: +5 (7 → 12)"

### Сценарий 3: Получение опыта и уровня
1. Игрок побеждает врага
2. Получает 50 XP и новый уровень
3. ✅ В начале ответа AI: "⭐ XP: +50 (0 → 50)" и "🎯 Уровень повышен: 1 → 2!"

### Сценарий 4: Создание персонажа
1. Игрок создаёт персонажа
2. ✅ Первый блок UPDATE_STATS отправляется без уведомления (нет предыдущих статов)

## Примечания

1. **Уведомления показываются только при изменении** — если HP не изменились, уведомления не будет

2. **Сравнение с предыдущими статами** — если игрок перезаходит, система всё равно корректно покажет изменения

3. **Форматирование** — уведомления используют Markdown (**жирный**) для заголовка

4. **Разделители** — линия "━━━" визуально отделяет уведомления от основного текста
