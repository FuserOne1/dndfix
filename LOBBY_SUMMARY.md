# 🎭 Система лобби - Итоговая сводка

## 🎉 Что реализовано

Полная система лобби с выбором/созданием персонажей и блокировкой занятых героев.

## 🎯 Ключевые возможности

### 1. Выбор персонажа
```
┌─────────────────────────────────────┐
│ Список персонажей                   │
│                                     │
│ ┌─────────┐  ┌─────────┐           │
│ │ Торин   │  │ Элара   │ 🔒       │
│ │ Дварф   │  │ Эльф    │ Занят    │
│ │ Воин    │  │ Следопыт│           │
│ │ [Выбрать]│  │         │           │
│ └─────────┘  └─────────┘           │
│                                     │
│ [+ Создать персонажа]               │
└─────────────────────────────────────┘
```

### 2. Создание персонажа
- Имя (уникальное)
- Раса (7 вариантов)
- Класс (8 вариантов)
- Предыстория (опционально)
- Автогенерация характеристик (10-16)
- Автогенерация HP на основе Constitution

### 3. Блокировка персонажей
```sql
-- Constraint в БД
UNIQUE(lobby_id, character_id)

-- Один персонаж не может быть занят дважды в одном лобби
```

### 4. Комната ожидания
```
┌─────────────────────────────────────┐
│ Лобби: ABC123          👥 2/4       │
├─────────────────────────────────────┤
│ Участники:                          │
│                                     │
│ 👤 Торин (Дварф Воин) 👑 ✅ Готов  │
│ 👤 Элара (Эльф Следопыт) ⏳ Не готов│
│ ⏳ Ожидание игрока...               │
│ ⏳ Ожидание игрока...               │
│                                     │
│ [✅ Готов к игре]  [▶️ Начать игру] │
└─────────────────────────────────────┘
```

### 5. Heartbeat система
- Каждые 30 секунд обновляется `joined_at`
- Функция `cleanup_inactive_participants()` освобождает персонажей через 5 минут
- Автоматическое освобождение при закрытии вкладки


## 📊 Структура БД

```
characters (глобальные)
  ├─ id: UUID
  ├─ name: TEXT UNIQUE ← Блокировка по имени
  ├─ race, class, level
  ├─ hp_current, hp_max, xp
  ├─ strength, dexterity, constitution
  ├─ intelligence, wisdom, charisma
  ├─ background, equipment
  └─ story_summary, avatar_url

lobbies (комнаты ожидания)
  ├─ id: TEXT
  ├─ name: TEXT
  ├─ max_players: INTEGER (default 4)
  ├─ is_active: BOOLEAN
  ├─ created_by: TEXT
  └─ started_at: TIMESTAMPTZ

lobby_participants (связь)
  ├─ id: UUID
  ├─ lobby_id → lobbies(id)
  ├─ character_id → characters(id)
  ├─ user_session_id: TEXT ← Отслеживание сессии
  ├─ joined_at: TIMESTAMPTZ ← Heartbeat
  ├─ is_ready: BOOLEAN
  └─ UNIQUE(lobby_id, character_id) ← Блокировка персонажа

rooms (игровые комнаты)
  ├─ id: TEXT
  ├─ lobby_id → lobbies(id) ← Связь с лобби
  └─ character_stats: JSONB
```

## 🔒 Безопасность

### 1. Блокировка персонажей
```sql
-- На уровне БД
UNIQUE(lobby_id, character_id)

-- На уровне приложения
const { data: existing } = await supabase
  .from('lobby_participants')
  .select('id')
  .eq('lobby_id', lobbyId)
  .eq('character_id', characterId)
  .single();

if (existing) {
  throw new Error('Персонаж уже занят!');
}
```

### 2. Автоочистка
```sql
-- Функция очистки
CREATE OR REPLACE FUNCTION cleanup_inactive_participants()
RETURNS void AS $$
BEGIN
  DELETE FROM lobby_participants
  WHERE joined_at < NOW() - INTERVAL '5 minutes'
  AND lobby_id IN (
    SELECT id FROM lobbies WHERE is_active = true
  );
END;
$$ LANGUAGE plpgsql;
```

### 3. RLS политики
```sql
-- Все могут читать
CREATE POLICY "Anyone can view characters" ON characters
  FOR SELECT USING (true);

-- Все могут создавать
CREATE POLICY "Anyone can create characters" ON characters
  FOR INSERT WITH CHECK (true);
```
