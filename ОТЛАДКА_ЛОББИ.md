# 🔍 Отладка проблемы с отображением в лобби

## Шаг 1: Откройте консоль браузера

1. Нажмите **F12** (или правая кнопка → Inspect)
2. Перейдите на вкладку **Console**
3. Очистите консоль (иконка 🚫)

## Шаг 2: Попробуйте войти в лобби

1. Создайте лобби
2. Выберите персонажа
3. Смотрите что пишется в консоли

### Что должно быть в консоли:

```
=== JOINING LOBBY ===
Lobby ID: ABC123
Character ID: uuid-здесь
Character Name: Имя персонажа
User Session ID: session_...
Successfully inserted participant: [...]
Verification query result: [...]

=== LOBBY ROOM MOUNTED ===
Lobby: {...}
Character: {...}
User Session ID: session_...
Fetched participants: [...]
My session ID: session_...
My character: Имя
Found my participant: {...}
```

### Если видите ошибки:

Скопируйте ВСЁ из консоли и пришлите мне!

---

## Шаг 3: Проверьте данные в Supabase

Выполните этот SQL в Supabase SQL Editor:

```sql
-- Все участники лобби
SELECT 
  lp.id,
  lp.lobby_id,
  lp.character_id,
  lp.user_session_id,
  lp.is_ready,
  c.name as character_name
FROM lobby_participants lp
LEFT JOIN characters c ON c.id = lp.character_id
ORDER BY lp.joined_at DESC
LIMIT 10;
```

Пришлите результат!

## Шаг 4: Используйте кнопку "Обновить"

В лобби теперь есть кнопка **"Обновить"** рядом с заголовком "Участники".

Нажмите её и посмотрите что изменится в консоли.

## Возможные проблемы:

### 1. Запись не создается в БД
**Симптом:** В консоли ошибка при `insert`
**Решение:** Проверьте что выполнили SQL скрипт для отключения RLS

### 2. Запись создается, но не загружается
**Симптом:** `Successfully inserted` есть, но `Fetched participants` пустой
**Решение:** Проблема с JOIN запросом или правами на чтение

### 3. Разные session ID
**Симптом:** `My participant not found in list!`
**Решение:** Session ID не совпадает при создании и чтении

## Быстрый тест в SQL:

```sql
-- Создаем тестовую запись
INSERT INTO lobby_participants (
  lobby_id,
  character_id,
  user_session_id,
  is_ready
) VALUES (
  'TEST',
  (SELECT id FROM characters LIMIT 1),
  'test_session_123',
  false
);

-- Проверяем что она видна
SELECT * FROM lobby_participants WHERE user_session_id = 'test_session_123';

-- Удаляем тест
DELETE FROM lobby_participants WHERE user_session_id = 'test_session_123';
```

Если этот тест работает - значит проблема в коде приложения.
Если не работает - значит проблема с правами в БД.
